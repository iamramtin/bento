This component is experimental and therefore subject to change or removal outside of major version releases.

Cache the output of one or more processors. This is especially useful when processing messages is expensive.

Introduced in version 3.64.0.

```yml
# Config fields, showing default values
label: ""
cached:
  cache: ""
  key: ""
  ttl: ""
  processors: []
```

## Fields

### `cache`

The cache resource to read and write processor results from.


Type: `string`  

### `key`

The cache key to use for a given message.
This field supports [interpolation functions](/docs/configuration/interpolation#bloblang-queries).


Type: `string`  

### `ttl`

The expiry period for each cache entry.


Type: `string`  

### `processors`

The list of processors whose result will be cached.


Type: `array`
